## Overview

The Init plugin provides an interactive wizard to generate a `tfui.hcl` configuration file, similar to `npm init`. It detects the terraform binary, scans for common monorepo project patterns, and writes a config file to the current directory.

## Usage

Press `i` from the home screen, or run `tfui init` from the CLI.

### Interactive (TUI)

The wizard progresses through these states:

1. **Detecting** -- scans the filesystem for terraform projects
2. **Review** -- shows detected patterns; toggle them on/off
3. **Confirm** -- previews the YAML that will be written
4. **Done** -- file written successfully

| Key | Action |
|-----|--------|
| `j` / `k` | Navigate detected paths |
| `Space` | Toggle a path on/off |
| `Enter` | Proceed to next step |
| `Esc` | Cancel / go back |

### Non-interactive (CLI)

```bash
tfui init --dir ./my-project
```

This runs detection and writes `tfui.hcl` without prompting.

## Detection Logic

The scanner checks for these common monorepo layouts:

- `modules/*` -- directories containing .tf files
- `envs/*` -- environment directories containing .tf files
- `infra/*` -- infrastructure directories containing .tf files
- `services/*/terraform` -- service-specific terraform directories
- `.` -- root directory .tf files

It also detects whether `tofu` or `terraform` is available on PATH.

## Output

```yaml
# Generated by tfui init
terraform_binary: terraform

projects:
  paths:
    - "modules/*"
    - "envs/*"
```
